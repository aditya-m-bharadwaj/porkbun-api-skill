# Build the CLI (v0.1.0-alpha.1)

This session converted the stub at `bin/porkbun-api-skill` into a working
single-file CLI implementing the full SKILL.md contract. The scaffold from
the earlier 2026-05-11 session set up the directory layout, AI prompts, ADRs,
and graphify integration; this session filled in the executable behavior.

Sister-project reference: [`linode-api-skill`](https://github.com/aditya-m-bharadwaj/linode-api-skill).
Architecture mirrors it (monolithic file, stdlib-only, OS keystore, AI-safe
credential entry via GUI, audit log, classifier-gated mutations), adapted for
Porkbun's dual-key auth, idempotency-key requirement, and registrar-domain
semantics.

## What was done

### CLI implementation (`bin/porkbun-api-skill`, ~1000 LOC)

Sections, in file order, with comment banners between each:

1. **Module docstring, imports, version constant, regexes, paths.** `VERSION = "0.1.0-alpha.1"`. `API_BASE = "https://api.porkbun.com/api/json/v3"`. `KEYSTORE_SERVICE = "porkbun-api-skill"`. Domain / subdomain / DNS-type / path / record-id regexes.
2. **`CtlError`** — user-facing error type whose message is safe to print.
3. **Platform helpers** — `_platform`, `_which`, `_account` (carry-over).
4. **Credential store (dual-key).** `_pack`/`_unpack` for the JSON blob `{"apikey": ..., "secretapikey": ...}`. macOS Keychain via `security`, Linux Secret Service via `secret-tool`, file fallback at `~/.porkbun-api-skill/credentials.json` (mode 0600). `creds_get` / `creds_set` / `creds_delete` operate on the pair atomically — either both or neither. Env-var fallback reads `PORKBUN_API_KEY` + `PORKBUN_SECRET_API_KEY`.
5. **GUI dialog helpers.** Two-step osascript on macOS (one `display dialog` per key, with hidden answer). `zenity --forms --add-password` (two fields, `\x1f` separator) on Linux, with kdialog fallback (two sequential `--password` prompts). `Get-Credential` on Windows with the convention `UserName=apikey, Password=secretapikey`. `_confirm_gui` for replace-existing prompts.
6. **HTTP layer (`_request`).** Single entry point. Auto-attaches `X-API-Key` / `X-Secret-API-Key` from the keystore. Auto-attaches `Idempotency-Key: <uuid4>` on every POST unless `attach_idempotency=False` is passed (which only happens via the explicit `--no-idempotency-key` flag on the generic `api` command). Strips any echo of the secret keys from error response bodies before raising. Returns parsed JSON, or `{"_raw": ...}` if the response is non-JSON.
7. **Pagination (`_paginate_listall`).** Iterates `/domain/listAll` in 1000-domain chunks via the `start` offset. Hard ceiling at 100k to guard against runaway loops.
8. **Audit log (`_audit`).** Appends one JSON line to `~/.porkbun-api-skill/audit.log` (mode 0600) with timestamp, user, action, target, idempotency key, classification, and body-key list. Never logs values, never logs credentials.
9. **Safety classifier.** Five tiers: `read`, `mutating`, `destructive`, `billable`, `privilege`. Templates have `{domain}`/`{subdomain}`/`{id}`/`{type}`/`{keytag}` placeholders. `_match_template` and `_normalize_path` walk templates longest-first so the most specific match wins. The `_READ_POST` set explicitly marks read-style POSTs (`/ping`, `/pricing/get`, `/domain/checkDomain/{domain}`, `/dns/retrieve*`, `/domain/get*`, `/marketplace/getAll`, `/account/inviteStatus`, `/domain/listAll`) so they don't fall through to `mutating`. POST-to-`/delete*` routes are explicit-only on `_DESTRUCTIVE_EXACT` — there is no method-based default for destructive (POST is the only verb Porkbun uses for delete). SSL retrieve is `privilege` on *both* GET and POST because the response body contains private key material. `_strip_v3` handles `/v3/...` and `/api/json/v3/...` user-typed prefixes safely (does NOT match `/v30/...`).
10. **Helpers.** `_validate_domain`, `_validate_subdomain_label`, `_parse_price_to_cents` (string `"9.73"` → int `973`), `_emit` (text vs `--json`), `_format_summary` (domain rows and DNS-record rows), `_load_body` (`--data` / `--body @file.json`, refuses paths inside `~/.porkbun-api-skill/`), `_parse_query` (`k=v` pairs).
11. **Named commands.**
    - `setup` — TTY-only `getpass` x2; validates via `POST /ping` before storing.
    - `gui-setup` — AI-runnable; native dialog flow.
    - `uninstall-credentials --yes` — clears all credential storage backends.
    - `whoami` — `POST /ping` with empty body; reports `credentialsValid` + `yourIp`.
    - `balance` — `GET /account/balance`.
    - `domains [--expiring-within-days N] [--name-contains S] [--tlds T]` — paginated listAll.
    - `domain <name>` — `GET /domain/get/{domain}`.
    - `price <name>` — `POST /domain/checkDomain/{domain}`; prints `--cost-cents N` hint for `register`.
    - `register <name> --cost-cents N --yes --i-understand-billing [--idempotency-key K]` — re-fetches checkDomain and refuses if `response.price` != `--cost-cents`.
    - `renew <name> --cost-cents N [--years Y] --yes --i-understand-billing` — same price-confirm-check against `response.additional.renewal.price`.
    - `transfer <name> --cost-cents N --auth-code C --yes --i-understand-billing` — checks `response.additional.transfer.price`; redacts `authCode` in `--dry-run`.
    - `auto-renew <name> --on|--off --yes` — `POST /domain/updateAutoRenew`.
    - `nameservers <name> --get | --set ns1,ns2,... --yes` — validates each NS hostname; refuses fewer than 2.
    - `dns list <name>` — `GET /dns/retrieve/{domain}`, prints record rows.
    - `dns add <name> --type T --content C [--subdomain S] [--ttl N] [--prio P] [--notes ...] --yes`.
    - `dns edit <name> --id ID --confirm-id ID --type T --content C [...] --yes` — refuses if `--confirm-id` != `--id`.
    - `dns delete <name> --id ID --confirm-id ID --yes`.
    - `ssl <name> --yes --allow-privilege` — writes timestamped triplet to `~/.porkbun-api-skill/ssl/<domain>.<ts>.{certificatechain,privatekey,publickey}.pem` (mode 0600). Never echoes key material to chat/stdout; only the file paths.
    - `audit-log [--last N]`.
    - `classify <METHOD> <path>` — plan-time helper.
12. **Generic `api` command.** Honors the same flag matrix as named commands. Accepts only GET / POST (Porkbun has no DELETE/PUT). For billable raw calls, re-runs `_verify_price` as a safety belt and injects `cost`/`agreeToTerms` into the body if missing. `--idempotency-key K` overrides the auto-UUIDv4; `--no-idempotency-key` opts out (debug only).
13. **Argparse / `build_parser()` / `main()`.** `CtlError` -> exit 2; KeyboardInterrupt -> exit 130.

### Tests (`tests/test_classify.py`, 52 tests)

Expanded from the single placeholder to a full offline suite. Classes:

- `TestNormalizePath` — known templates, `/v3` and `/api/json/v3` prefix stripping, `/v30` does NOT match `/v3`, unknown paths pass through.
- `TestClassify` — every tier (read / mutating / destructive / billable / privilege), `confirm-name` vs `confirm-id` distinction, unknown POST defaults to mutating (deliberately NOT destructive — operators must register delete-style routes explicitly), case-insensitive method names, `/v3` and `/api/json/v3` prefix handling.
- `TestPathValidation` — traversal and metacharacter rejection.
- `TestDomainValidation` — accepts xn-- / multi-label / .co.uk, rejects garbage.
- `TestPriceParsing` — `"9.73"` → 973, `"$9.73"` → 973, `"9.7"` → 970, rejects garbage.
- `TestPlatformHelpers` — type-only smoke.
- `TestPackUnpack` — JSON blob round-trip.
- `TestCliEntrypoint` — `--version` exits 0; `classify` runs without creds.

All 52 pass on macOS / Python 3.13 in ~7 ms.

### Bug fixed during this session

`_strip_v3` initially used `path.startswith("/v3")` (no trailing slash), which
matched `/v30/foo`. Fixed to use literal prefixes including the trailing `/`,
with separate equality checks for the bare-prefix-only case. Caught by
`TestNormalizePath.test_does_not_strip_unrelated_prefix`. Same shape of bug
that the sister project had at the same stage — the test ports over.

## Open questions (carried + new)

Carried from scaffold session (still TBD):

- **SSL bundle output path.** Currently timestamped — every retrieve adds a
  triplet; the user has to clean up. Alternative: overwrite-latest. Decided
  to keep the current timestamped approach for now (history is useful for
  rotation auditing); revisit if the directory grows unmanageably.
- **`/account/invite` classification.** Kept as `privilege` for this build.
  Will revisit if the response shape shows the invitee gets isolated
  credentials (in which case `mutating` may suffice).
- **Idempotency-Key on read-style POSTs.** Current design always attaches
  (simpler `_request`; Porkbun's docs say the header is ignored on read
  endpoints). Confirmed during implementation; not changing.

New questions from this session:

- **Pagination response shape for `/domain/listAll`.** The spec doesn't
  explicitly document the response wrapper. `_paginate_listall` assumes
  `{"domains": [...]}` based on the inferred convention; live validation
  needs to confirm. If the actual key is different, `cmd_domains` needs a
  one-line patch.
- **DNS retrieve response wrapper.** `cmd_dns_list` assumes
  `{"records": [...]}`. Same caveat — confirm with live data.
- **Audit log redaction policy for the `api` command.** When a caller passes
  `--data` with a body containing what *might* be a credential (e.g. another
  endpoint's API key field), the audit log records only the body's *keys*,
  not values. Sufficient defense; worth a CHANGELOG note when the public
  release lands.
